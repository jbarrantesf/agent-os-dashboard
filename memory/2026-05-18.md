# Daily Memory — 2026-05-18

## 🔧 Fix: Google Meet plugin (`openclaw googlemeet join`)

### Síntoma
- Comando `openclaw googlemeet join <url>` ya no existía (funcionaba en 26-abr).
- Plugin `google-meet` aparecía como "loaded" pero el CLI host no exponía el subcomando.

### Root cause
**Desfase de versiones CLI ↔ plugin**:
- CLI host instalado vía npm: `2026.5.12` (de hace ~6 días)
- Plugin `@openclaw/google-meet` publicado HOY: `2026.5.18`
- Plugin declara `peerDependencies: openclaw >=2026.5.18` y `compat.pluginApi >=2026.5.18`
- El plugin nuevo usa una API que el CLI viejo no entiende → carga pero no registra subcomandos.

### Fix
```bash
npm install -g openclaw@latest
# Luego restart del gateway
openclaw gateway restart
```
- CLI subió `2026.5.12 → 2026.5.18 (50a2481)`
- Plugin `google-meet 2026.5.18` ahora expone: `join`, `create`, `auth`, `doctor`, `attendance`, `artifacts`, `calendar-events`, `end-active-conference`, `export`

### Verificación
- `openclaw googlemeet doctor` → "Google Meet sessions: none" (sano)
- `openclaw googlemeet auth login` → disponible para re-auth si hace falta
- `openclaw googlemeet join <url>` → subcomando registrado ✅

### Lección operativa
- **Cuando `openclaw plugins install <X>` deja un plugin que "carga" pero el comando no aparece** → casi siempre es desfase CLI/plugin.
- Verificar con: `openclaw --version` vs `cat ~/.openclaw/npm/node_modules/@openclaw/<plugin>/package.json | grep version`
- Si el plugin pide `compat.pluginApi` mayor a la CLI → `openclaw update` o `npm install -g openclaw@latest`.
- **NO usar `openclaw update --yes` cuando estoy corriendo dentro del gateway que va a reiniciar** — me corta a mí mismo. Mejor `npm install -g openclaw@latest` + `openclaw gateway restart` agendado en background.

### Reauth Google Meet (completado 15:02 hrs)
- Token viejo daba `invalid_grant` desde hoy en la mañana.
- Comando: `openclaw googlemeet auth login` → abre URL OAuth en browser + levanta server en `localhost:8085`.
- ⚠️ Detalle clave: el comando **imprime el bloque JSON con los tokens nuevos pero NO escribe automáticamente en config**. Hay que copiar/pegar a mano (o yo lo hago vía Python).
- ⚠️ **Schema del plugin solo acepta `oauth.{clientId, clientSecret, refreshToken}`**. Si incluís `accessToken`, `expiresAt`, `scope`, `tokenType` → da `must NOT have additional properties` y el comando `googlemeet` desaparece del CLI hasta limpiar.
- ✅ Verificado: refresh válido por 3599s, scopes Meet+Calendar+Drive OK.
- Backup pre-cambio: `~/.openclaw/openclaw.json.bak-pre-oauth-<timestamp>`.
